The Problem

Sorting 9,564 possible planets from noise

NASA's Kepler telescope watched 150,000 stars for 9 years, logging every tiny brightness dip. Most dips are not planets — they are eclipsing binary stars, cosmic rays, or instrument noise. Each candidate needs expensive ground-based follow-up to confirm. Celesta is a first-pass classifier that reads raw photometry and predicts which candidates are worth investigating.

⚠️
Leakage-free constraint
The NASA archive includes koi_pdisposition and koi_score — NASA's own pre-classification and confidence score. Using them gives 99%+ accuracy but is scientifically worthless (we'd be copying NASA's answer). Celesta uses only raw telescope measurements: transit photometry and stellar spectroscopy — no vetting outputs.

Feature Engineering

12 physics-motivated features

Raw measurements tell the model what was recorded. Engineered features tell it what those measurements mean physically. All 12 are derived from Kepler photometry and stellar catalogues — no external data.

All 12 Features

#FeaturePhysical Meaning
1single_multi_ratioOne-off event vs repeating transit — the false positive fingerprint
2duration_period_ratioTransit chord encodes stellar density via Kepler's third law
3log_periodLog-compresses 3-order-of-magnitude range for tree splits
4log_depthSame — transit depth is heavily right-skewed
5log_snrSame

7impact_ror_ratioNear 1 → grazing transit → eclipsing binary signature
8expected_duration_ratioObserved vs Keplerian duration — eccentricity / blended EB flag
9snr_per_transit newPer-transit SNR — consistent for real planets, noisy for artefacts

Model Architecture

Soft-voting ensemble of three complementary models

No single algorithm dominates every aspect of this dataset. Averaging probability outputs from three models with different inductive biases cancels individual errors.
